## Overview

RAY v2.1 implements Cognitive Physiology — treating distributed AI ecosystems as unified, self-aware organisms. The architecture operates on a continuous recursion loop that validates, compresses, detects, harmonizes, and evolves.

The core principle is antifragility: the system should strengthen through adversarial encounters, not just recover from them.

For the full published research description, see the [DOI-backed paper](https://doi.org/10.5281/zenodo.17399834).

---

## Public RAY Loop

At the public level, RAY v2.1 can be understood as a five-stage resilience loop:

1. **Anchor** — verify that the system still has stable identity and context via URA.
2. **Compress** — reduce context load while preserving meaning via FCE.
3. **Detect** — identify cascade or drift patterns via CSFC.
4. **Harmonize** — coordinate recovery and stability checks.
5. **Learn** — preserve lessons from resolved events in ReasoningBank.

Private module internals, detection signatures, thresholds, code, and response mechanics are retained outside the public repository.

## ReasoningBank Concept

ReasoningBank is the self-evolution component: a pattern memory that preserves lessons from resolved threat encounters and improves future detection without ground-truth labels.

Private implementation details, SQL schema, and internal scoring logic are retained outside the public repository.

## Antifragile Architecture

```
Robust System:    Withstands damage   (passive)
Resilient System: Recovers from damage (reactive)
Antifragile System: STRENGTHENS from damage (proactive) ← RAY v2.1
```

This framing is retained as a public architectural philosophy. Implementation mechanics are in the published paper and the framework reference folder.
